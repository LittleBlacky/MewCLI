## 1. 设计目标

**Tool 工具系统** - 统一的工具注册、执行、权限控制机制，为 Agent 提供能力扩展。

### 1.1 设计原则

| 原则         | 说明                       | 原因                   |
| ------------ | -------------------------- | ---------------------- |
| **统一注册** | 所有工具通过 Registry 注册 | 集中管理               |
| **权限可控** | 危险命令需要确认           | 安全保障               |
| **可扩展**   | 支持 MCP 插件扩展          | 功能扩展               |
| **可观测**   | 工具调用可追踪             | 便于调试               |
| **异步执行** | 工具执行不阻塞事件循环     | 支持多 Worker 真正并行 |

### 1.2 为什么这样设计

| 设计决策      | 原因             | 好处           |
| ------------- | ---------------- | -------------- |
| Registry 模式 | 集中管理所有工具 | 便于查询和配置 |
| 权限分层      | 危险命令单独处理 | 安全可控       |
| MCP 集成      | 支持外部系统     | 能力扩展       |
| 工具分类      | 按功能分类组织   | 便于理解       |

## 2. 模块结构

```
tool/
├── __init__.py              # 导出公共类型
├── registry.py             # 工具注册表（自动注册 + MCP 动态加载）
├── builtin/                 # 内置工具（@auto_tool 装饰的函数）
│   ├── file.py             # 文件操作
│   ├── bash.py             # Bash 执行
│   ├── search.py           # 搜索
│   └── ...
├── permission.py           # 权限控制
└── mcp.py                  # MCP 集成
```

> **设计原则**：ToolNode 执行，Hook 控制权限，Registry 管理工具，@auto_tool 自动注册。

---

## 3. 核心组件

| 组件                  | 职责           | 选用原因                     |
| --------------------- | -------------- | ---------------------------- |
| **ToolNode**          | 工具执行       | LangGraph 内置，无需自己实现 |
| **Hook (PERMISSION)** | 执行前权限检查 | 优先级 100，可拦截危险命令   |
| **ToolRegistry**      | 工具管理       | MCP 动态注册/注销需要        |
