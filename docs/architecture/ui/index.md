## 1. 设计目标

**UI 交互** - 提供用户与系统交互的界面，包括 REPL 交互和 CLI 命令行界面。

### 1.1 设计原则

| 原则 | 说明 | 原因 |
|------|------|------|
| **简洁交互** | 最小化用户操作 | 提高效率 |
| **即时反馈** | 操作后立即反馈 | 良好体验 |
| **可扩展** | 支持自定义命令 | 灵活扩展 |
| **可配置** | 输出格式可配置 | 适应需求 |

### 1.2 为什么这样设计

| 设计决策 | 原因 | 好处 |
|----------|------|------|
| REPL 模式 | 交互式对话体验 | 直观方便 |
| 流式输出 | 即时看到结果 | 体验好 |
| 多命令支持 | / 开头的命令系统 | 功能完整 |
| 分页输出 | 处理长输出 | 可读性好 |

---

## 2. 模块结构

```
ui/
├── __init__.py              # 导出公共类型
├── repl.py                  # REPL 交互界面
└── cli.py                   # CLI 入口
```

> **说明**：UI 层位于顶层目录 `ui/`，与 `core/`、`infra/` 等平级，负责用户交互界面。

---

## 3. 核心组件

| 组件 | 职责 | 说明 |
|------|------|------|
| **REPL** | 交互式对话 | 核心交互界面 |
| **CLI** | 命令行入口 | 程序入口 |
| **OutputFormatter** | 输出格式化 | 文本、代码、表格格式化 |
| **StreamingOutput** | 流式输出 | 即时响应 |
