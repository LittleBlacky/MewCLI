# MiniCode 需求分析文档

> 版本: v2.0
> 日期: 2026-05-03
> 状态: 进行中

---

## 1. 产品愿景

**一个可自我进化的多 Agent 智能终端助手**

| 关键词           | 含义                             |
| ---------------- | -------------------------------- |
| **可自我进化**   | Agent 能够从经验中学习，自动改进 |
| **多 Agent**     | 支持多个 Agent 协作（主从模式）  |
| **智能终端助手** | 在终端环境中运行                 |

---

## 2. 需求概览

### 2.1 核心功能选择

| 维度     | 选择         | 说明                                                  |
| -------- | ------------ | ----------------------------------------------------- |
| 协作模式 | 主从模式     | Lead 理解任务，分解子任务；Worker 并行执行；Lead 汇总 |
| 终端范围 | 受限模式     | 大部分命令允许，危险命令需确认（如 `rm -rf`）         |
| 外部集成 | MCP 动态集成 | 通过 MCP 插件支持任意外部系统                         |
| 运行环境 | 本地单进程   | 所有 Agent 在同一进程运行                             |

### 2.2 自我进化选择

| 维度         | 说明                                       |
| ------------ | ------------------------------------------ |
| **偏好学习** | 被动观察用户行为、主动询问、显式配置       |
| **知识学习** | 启动时扫描项目、任务中按需学习、用户注入   |
| **技能创建** | 高质量任务自动生成、用户手动触发、模板驱动 |

### 2.3 技术决策

| 维度      | 选择         | 说明                                |
| --------- | ------------ | ----------------------------------- |
| LangGraph | 核心框架     | 作为核心的状态管理和流程控制框架    |
| MVP 范围  | 多 Agent MVP | Lead + N 个 Worker，动态分配        |
| 记忆实现  | 三层记忆     | Preference + Knowledge + Episodic   |
| 重构策略  | 全新设计     | 删除旧代码，重新设计目录结构        |
| 目录结构  | 按职责分     | core/agent、core/memory/、infra/ 等 |
| 任务分解  | 智能分解     | Agent 决定怎么分解，用 LLM 推理     |
| 进化时机  | 实时进化     | 任务完成后立即分析并学习            |
| Prompt 缓存 | 支持多厂商   | 静态内容缓存降低成本（详见架构文档）|

### 2.4 核心概念定义

| 抽象          | 职责                           | 生命周期        |
| ------------- | ------------------------------ | --------------- |
| **Session**   | 用户会话，管理上下文           | 用户开始到结束  |
| **Team**      | 管理多个 Agent 的协作          | Session 内      |
| **Agent**     | 智能执行单元（Lead 或 Worker） | Task 开始到结束 |
| **Task**      | 子任务描述                     | 分配到完成      |
| **Evolution** | 从经验中学习、创建技能         | 持续运行        |

```
│                   Memory Layer                              │
│        (Preference / Knowledge / Episodic)                   │
│                          +                                   │
│                   Dream（控制层）                            │
├─────────────────────────────────────────────────────────────┤
│                   Evolution Layer                           │
│              (Engine / Skill Synthesizer)                    │
└─────────────────────────────────────────────────────────────┘
```

**State 设计**：

```
GraphState (infra/graph.py - 基础设施层)
    └── AgentState (core/agent/state.py - Agent 层，使用 add_messages)
```

### 3.3 核心数据流

```
User Input
    │
    ▼
┌─────────────────┐
│    REPL/CLI     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐     ┌────────────────┐
│ Session Manager │────►│    Hooks       │  ← 生命周期钩子
└────────┬────────┘     └────────────────┘
         │
         ▼
┌─────────────────┐     ┌────────────────┐
│   Lead Agent    │────►│  Skill Match   │  ← 任务匹配技能
│   (理解+分解)   │     └────────────────┘
└────────┬────────┘
         │ 分解任务
         ▼
┌─────────────────┐     ┌────────────────┐
│  Team Manager   │────►│   Workers      │  ← 并行执行
│  (分配任务)     │     └────────────────┘
└────────┬────────┘
         │ 结果
         ▼
┌─────────────────┐     ┌────────────────┐
│   Lead Agent    │────►│  Memory Layer  │  ← 记忆更新
│   (汇总结果)    │     └────────────────┘
└────────┬────────┘
         │
         ▼         ◄──────── Dream（会话间隙自动触发）
┌─────────────────┐
│   Evolution     │     ┌────────────────┐
│     Engine      │────►│  Skill Synth   │  ← 技能生成
└─────────────────┘     └────────────────┘
         │
         ▼
    User Output
```

### 3.4 Lead / Worker Agent 职责

| Agent            | 职责                                                                   |
| ---------------- | ---------------------------------------------------------------------- |
| **Lead Agent**   | 理解用户意图、分解任务为子任务、分配给 Workers、监控执行状态、汇总结果 |
| **Worker Agent** | 接收子任务、使用工具执行、报告执行结果、参与自我进化                   |

## 4. 核心模块设计

### 4.1 记忆系统（Memory Layer）

| 层级           | 内容                              | 持久化 | 模块位置                    |
| -------------- | --------------------------------- | ------ | --------------------------- |
| **Preference** | 用户偏好（tab vs 空格、常用命令） | 长期   | `core/memory/preference.py` |
| **Knowledge**  | 项目知识（结构、技术栈）          | 长期   | `core/memory/knowledge.py`  |
| **Episodic**   | 事件记忆（过去发生的事）          | 短期   | `core/memory/episodic.py`   |

### 4.2 Dream（梦境整合 - 控制层）

Dream 是控制层，在会话间隙自动触发，对记忆进行去重和优化。

| 触发方式 | 深度   | 说明         |
| -------- | ------ | ------------ |
| 会话结束 | light  | 快速去重     |
| 定时     | normal | 每 30 分钟   |
| 阈值     | deep   | 记忆数 > 100 |

**门控条件**：冷却时间(24h)、最小会话数(5)、并发锁

### 4.3 Skill（技能系统 - 独立模块）

| 组件              | 说明       | 位置                     |
| ----------------- | ---------- | ------------------------ |
| **SkillRegistry** | 技能注册表 | `core/skill/registry.py` |
| **SkillMatcher**  | 技能匹配器 | `core/skill/matcher.py`  |

Evolution Engine 调用 Skill 模块管理技能，而非自行处理。

### 4.4 Hook（通用生命周期钩子）

Hook 位于 `infra/hook/`，提供通用生命周期扩展点。

```python
class HookPriority:
    LOGGING = 10       # 日志记录
    METRICS = 20       # 指标收集
    PERMISSION = 100   # 权限检查
    SECURITY = 120     # 安全检查
    USER_HOOK = 300    # 用户自定义
    PLUGIN = 400       # 插件钩子
```

| Hook 类型     | 说明           |
| ------------- | -------------- |
| `TOOL_*`      | 工具执行前后   |
| `AGENT_*`     | Agent 生命周期 |
| `SESSION_*`   | 会话生命周期   |
| `EVOLUTION_*` | 进化生命周期   |

### 4.5 State 设计

```
GraphState (infra/graph.py)
    │
    ├── messages: list           # 基础消息
    ├── mode: str                # 运行模式
    └── ...
            │
            ▼ 组合
AgentState (core/agent/state.py)
    │
    ├── messages: Annotated[list, add_messages]  # 消息自动合并
    ├── task: Optional[Task]      # 当前任务
    ├── subtasks: list[Task]      # 子任务
    └── ...
```

**为什么不合并？** GraphState 是基础设施层通用定义，AgentState 需要 `add_messages` 实现消息自动合并。

### 4.6 权限系统

- 内置危险模式（不可覆盖）
- 用户可配置 allow/deny 规则
- 危险命令需确认

### 4.7 Evolution Engine（自我进化）

| 功能     | 说明                                          |
| -------- | --------------------------------------------- |
| 收集数据 | 收集任务执行数据                              |
| 模式判断 | 使用 LLM 判断（简化设计，无 PatternDetector） |
| 技能生成 | 调用 SkillSynthesizer 创建/更新技能           |
| 记忆更新 | 更新 Preference 和 Knowledge                  |

### 4.8 Session（会话上下文）

Session 管理用户会话的生命周期，包括上下文、状态追踪和持久化。

| 组件               | 说明                               | 位置                      |
| ------------------ | ---------------------------------- | ------------------------- |
| **SessionContext** | 会话上下文，存储消息历史、容量管理 | `core/session/context.py` |
| **SessionManager** | 会话生命周期管理                   | `core/session/manager.py` |

**核心功能**：

- **上下文隔离**：每个会话有独立的上下文，支持多会话并发
- **容量管理**：追踪 token 使用，自动触发压缩（阈值 70%）
- **状态持久化**：会话状态可保存和恢复，支持断点续连
- **记忆注入**：管理 Preference、Knowledge、Skill 的注入状态

### 4.9 Prompt 缓存

Prompt 缓存通过将静态内容（系统提示、知识库、技能）缓存以降低 API 成本。

| 可缓存内容 | 缓存 | 说明 |
|-----------|------|------|
| System Prompt | ✅ | 固定不变 |
| 项目知识 | ✅ | 相对稳定 |
| 技能定义 | ✅ | 很少变化 |
| 用户偏好 | ✅ | 长期有效 |
| 对话历史 | ❌ | 动态变化 |
| 当前输入 | ❌ | 每次不同 |

**多厂商支持**：

| 厂商 | 支持程度 | 机制 |
|------|----------|------|
| **Anthropic** | ✅ 完全支持 | `cache_breakpoint` + `cache_control: {type: "ephemeral"}` |
| **OpenAI** | ✅ 支持 | `cache_control` with `ephemeral` |
| **Google** | ⚠️ 部分支持 | 查看 Vertex AI 文档 |
| **其他** | ❌ 回退 | 缓存内容作为系统提示 |

**核心组件**：

| 组件 | 说明 | 位置 |
|------|------|------|
| **PromptCache** | 管理缓存内容构建 | `infra/cache/prompt.py` |
| **CacheBudgetManager** | 管理缓存 Token 预算（最大 200K） | `infra/cache/budget.py` |
| **CacheAdapter** | 厂商适配器（抽象基类） | 详见架构文档 |

**详细设计**：见 `docs/architecture/07_infrastructure.md` 第 4.4 节。

## 7. 量化指标体系

> 详细实现代码见 `docs/architecture/` 相关文档

### 7.1 指标分类

| 类别                   | 指标                           | 用途                |
| ---------------------- | ------------------------------ | ------------------- |
| **TeamMetrics**        | 任务分解、并行执行、通信效率   | 评估 Agent 协作效果 |
| **EvolutionMetrics**   | 技能创建、模式检测、学习效果   | 评估自我进化能力    |
| **PerformanceMetrics** | 响应时间、Token 使用、错误恢复 | 评估系统性能        |
| **UserMetrics**        | 使用频率、权限交互、用户纠正   | 评估用户体验        |
| **MemoryMetrics**      | 记忆命中率、压缩效果           | 评估记忆系统        |
| **ContextMetrics**     | 上下文容量、溢出恢复           | 评估上下文管理      |

### 7.2 量化

| 亮点           | 量化指标                                             | 目标                      |
| -------------- | ---------------------------------------------------- | ------------------------- |
| 多 Agent 协作  | 任务分解成功率、最大并行 Worker 数、并行节省时间     | 95%、4 个、60%+           |
| 自我进化引擎   | 技能库增长、技能命中率、系统改进评分                 | 15+ 个、35%+、持续上升    |
| 断点持久化     | Checkpoint 恢复成功率、上下文 Token 节省、错误恢复率 | 99%+、50%+、85%+          |
| 三层记忆系统   | 偏好命中率、知识命中率、记忆总量                     | 90%+、85%+、50+ 条        |
| 智能上下文压缩 | 压缩次数、平均压缩比、Token 节省、溢出恢复率         | 200+ 次、60%+、10M+、99%+ |

## 8. 附录

### 8.1 已决策项

| 决策项          | 选择             | 说明                                           |
| --------------- | ---------------- | ---------------------------------------------- |
| 任务超时机制    | 1 分钟           | 单任务执行超时                                 |

| Dream 触发时机  | 混合模式         | 会话结束 + 定时(30min) + 阈值(100 条)          |

| 约束           | 说明                                 |
| -------------- | ------------------------------------ |
